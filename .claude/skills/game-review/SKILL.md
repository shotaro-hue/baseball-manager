# Game Review スキル

SPEC.md・ROADMAP.md・実際のコードを照合し、ゲームの理想形との乖離を評価して優先度付きのアクションリストを生成するワークフロー。

## ワークフロー

Make a todo list for all the tasks in this workflow and work on them one after another.

### 1. SPEC.md を読み込む

`SPEC.md` を読み込み、以下を把握する:
- ゲームの設計目標・コンセプト
- 全機能の一覧と詳細仕様
- 数値バランスの設計意図
- ユーザー体験・ゲームプレイフローの理想像

### 2. ROADMAP.md を読み込む

`ROADMAP.md` を読み込み、各 Tier の実装状況を整理する:
- ✅ 完了済みの機能
- 🔶 部分実装の機能（何が足りないか）
- 🔴 未着手の機能
- 既知のバグ（B* タグ）
- 計画中の改善（E* タグ）

### 3. 実際のコードを確認

ROADMAP と SPEC を照合しながら、主要ファイルを確認する:

**エンジン層** (`src/engine/`):
- `simulation.js` — シミュレーションロジックが SPEC の仕様を満たしているか
- `player.js` — 選手老化・引退が実装されているか
- `sabermetrics.js` — セイバーメトリクスが正しく計算されているか
- `awards.js` — 表彰・記録システムが機能しているか

**UI 層** (`src/components/`):
- `Tabs.jsx` — 各タブの機能が SPEC 通りか
- `TacticalGame.jsx` — 試合操作の UI が完成しているか
- `DashboardTab.jsx` — ダッシュボードに必要な情報が揃っているか

**状態管理**:
- `src/App.jsx` — game flow が SPEC の設計に沿っているか

**定数**:
- `src/constants.js` — バランス数値が SPEC の設計意図に沿っているか

### 4. 評価レポートを生成

以下の構造で評価レポートを作成する:

---

## ゲームレビューレポート

### 現状サマリー
- 完成度: XX%（機能数ベース）
- 現在のフォーカス: Tier X

### 理想形との主な乖離

#### 未実装の重要機能（インパクト順）
1. **[機能名]** (Tier X) — 理由・ゲームへの影響
2. ...

#### 仕様と実装の乖離
- **[ファイル名:行番号]** — SPEC のXXに対して、実装ではYYになっている
- ...

#### ゲームバランス・UX の改善余地
- ...

### 優先度付き次アクションリスト

| 優先度 | カテゴリ | タスク | 期待インパクト |
|---|---|---|---|
| 🔴 高 | feature / bugfix / balance | 内容 | ゲームへの貢献 |
| 🟡 中 | ... | ... | ... |
| 🟢 低 | ... | ... | ... |

### 推奨する次の一手
> [最もインパクトが高く、今すぐ着手すべき1つのアクションを具体的に提案]

---

### 5. ユーザーへの報告

レポートを提示し、「次の一手」のスキル（`/game-feature`, `/game-bugfix`, `/game-balance`）を案内する。

## 注意事項

- コードを修正しない。あくまで評価・提案に徹する
- ROADMAP.md の更新のみコミットが必要な場合は軽量なコミットで済ませる
- 「理想のゲーム」を SPEC.md の内容に忠実に解釈する（ユーザーの意図を最大限反映）
