# Game UI 改善スキル

新規 UI ウィジェットの追加・UX フロー改善を実装し、コミット・プッシュするワークフロー。
ゲームロジックの変更は行わず、表示層（`src/components/`・`src/styles.css`）に集中する。

## ワークフロー

Make a todo list for all the tasks in this workflow and work on them one after another.

### 1. 改善対象を特定

ユーザーが対象を指定していない場合は、以下から候補を抽出する:
- `ROADMAP.md` の UI/UX 関連タスク（E* タグ・UX 改善メモ）
- `SPEC.md` の「ユーザー体験・ゲームプレイフローの理想像」セクション
- 現状の UI と SPEC の乖離（情報が見つけにくい、操作ステップが多いなど）

**対象カテゴリ:**
- 新規ウィジェット: グラフ（Recharts）、比較テーブル、サマリーカード、モーダルダイアログ
- UX フロー改善: 確認ステップの削減、情報の再配置、ローディング/エラーフィードバック

### 2. 要件定義

`SPEC.md` の UX 仕様と現状の UI コードを照合し、以下の要件サマリーを作成する:

| 項目 | 内容 |
|---|---|
| 改善の目的 | 何が不便か・どう良くなるか |
| 対象コンポーネント | 変更する既存ファイルと新規作成するファイル |
| 表示データ | どの state・props を使うか |
| 受け入れ条件 | 何が実現できれば完了か |
| 非対象 | ゲームロジック変更・スタイル微調整・コンポーネント分割 |

作成した要件サマリーをユーザーに提示し、**必ず承認を得てから**実装に進む。

### 3. 関連コードを調査

**UI 層** (`src/components/`):
- `Tabs.jsx` — 対象タブの現状実装
- `TacticalGame.jsx` — 試合操作 UI（試合系改善の場合）
- `Screens.jsx` — 画面切り替え

**スタイル**:
- `src/styles.css` — CSS 変数・チームカラーテーマ（新規コンポーネントでも既存変数を使う）

**データ**:
- `src/App.jsx` — 使用できる state を確認
- `src/engine/sabermetrics.js` — グラフ用データ加工が必要な場合

既存の Recharts 実装（`LineChart`・`BarChart` 等）を確認し、再利用できるパターンを把握する。

### 4. 実装

- 既存の CSS 変数（`--team-primary`・`--team-secondary` 等）を活用し、ハードコードしない
- ゲームロジック（RNG・シミュレーション計算）は UI 層に書かない
- Recharts コンポーネントは既存パターンに倣う
- モーダル・オーバーレイは既存実装があれば再利用する
- props・コールバックの命名は既存コンポーネントの規則に合わせる

### 5. ビルド・テスト確認

`.claude/skills/_shared/build-and-test.md` を Read ツールで読み込み、記載の手順を実行する。

### 6. ROADMAP.md を更新（該当あれば）

UI 改善に対応する ROADMAP のタスクがあれば 🔴 → ✅（または 🔶）に更新する。
対応するタスクがなければスキップする。

### 7. SPEC.md を更新

- **`CHANGELOG.md`**（必須）: 最新エントリを先頭に追加する

  ```
  ### YYYY-MM-DD — <UI改善の名称>（コミットハッシュ）

  **仕様本文への影響あり（§2・§15）** or **仕様本文への影響なし**

  - 実装内容の箇条書き
  ```

- **§15 UI改善計画**: 該当タスクのステータスを「計画策定済み」→「✅ 実装済み（コミットハッシュ）」に更新する
- **§2 ディレクトリ構成**: 新規コンポーネントを追加した場合のみ更新する
- **§3 画面フロー**: タブ構成・画面遷移が変わった場合のみ更新する

### 8. コミット・プッシュ・PR

`.claude/skills/_shared/commit-push-pr.md` を Read ツールで読み込み、記載の手順を実行する。コミットプレフィックスは `feat:` を使う。

## 完了時の報告

以下の形式でユーザーに報告する:

- 実装した UI 改善の概要
- 変更・追加したファイル一覧
- ビルド結果: ✅ 成功 / ‼️ 失敗（詳細）
- ROADMAP.md の更新内容（変更した場合のみ）
- SPEC.md の更新内容（CHANGELOG.md + 更新したセクション一覧）
- マージ完了: PR #番号
- 次に改善を推奨する UI 箇所（あれば）

## game-feature との使い分け

| | game-feature | game-ui |
|---|---|---|
| 新しいゲームロジックを追加する | ✅ | ✗ |
| 既存データの表示方法を改善する | ✗ | ✅ |
| 新規グラフ・ウィジェットを追加する | ✗ | ✅ |
| 操作フロー・UX を改善する | ✗ | ✅ |
| engine/ ファイルを変更する | ✅ | ✗ |
