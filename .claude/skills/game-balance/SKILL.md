# Game Balance 調整スキル

`src/constants.js` と `src/engine/simulation.js` のパラメータを分析し、ゲームバランスを調整してコミット・プッシュするワークフロー。

## ワークフロー

Make a todo list for all the tasks in this workflow and work on them one after another.

### 1. バランス調整の目的を確認

ユーザーが調整したい内容を確認する。指定がない場合は以下を尋ねる:
- どの要素が強すぎ / 弱すぎると感じているか
- どんなゲームプレイ体験を目指しているか
- 変更したい具体的な数値・挙動はあるか

### 2. 現在のパラメータを把握

以下のファイルを読み込み、関連するパラメータを把握する:

**`src/constants.js`** — ゲームバランス定数の主要な置き場:
- チーム定義・初期能力値範囲
- 年俸テーブル・契約条件
- ドラフト・トレード評価係数
- スタジアム収益・投資効果
- モラル・スカウト精度係数

**`src/engine/simulation.js`** — シミュレーション内の確率定数:
- 打球種別確率（ゴロ・フライ・ライナー）
- 三振・四球・本塁打の基礎確率
- 利き手マッチアップボーナス
- 疲労・スタミナ影響係数
- 守備・走塁の成功確率

**`src/engine/player.js`** — 選手成長・老化パラメータ:
- 年齢別成長曲線
- 老化ペナルティ係数
- 引退確率テーブル

### 3. SPEC.md でバランス設計意図を確認

`SPEC.md` の関連セクションを読み、以下を確認する:
- 設計が目指すゲームバランスの方向性
- 各パラメータの意図・根拠
- NPB のリアルデータとの対応関係

### 4. 調整案を策定

現在値と調整後の値を比較した表を作成する:

| パラメータ | 現在値 | 調整後 | 変更理由 |
|---|---|---|---|
| `PARAM_NAME` | `0.xx` | `0.xx` | 変更の根拠 |

- 変更は 1 度に 1〜3 項目に絞る（影響範囲の把握を容易にするため）
- マジックナンバーは避け、意味のある名前の定数として定義する
- 変更コメントを添える

調整案をユーザーに提示し、**必ず立ち止まって以下の3択を待つ**。承認なしに変更（ステップ5以降）へ進んではならない。

| ユーザーの反応 | 次のアクション |
|---|---|
| **承認**（「OK」「進めて」等） | ステップ5へ進む |
| **質問・修正依頼**（「もう少し大きく変えたい」「別のパラメータにしたい」等） | 回答・調整案を修正して再提示。承認が得られるまでこのループを繰り返す |
| **中断**（「やめる」「キャンセル」等） | 変更せずに終了する |

### 5. 変更を適用

`src/constants.js`（または `src/engine/simulation.js`）を編集する。
コメント例:
```js
// 本塁打確率を NPB 平均打率に合わせて微調整 (旧: 0.035)
const HR_BASE_PROB = 0.030;
```

### 6. ビルド・テスト確認

`.claude/skills/_shared/build-and-test.md` を Read ツールで読み込み、記載の手順を実行する。

### 7. SPEC.md を更新

- **`CHANGELOG.md`**（必須）: 最新エントリを先頭に追加する

  ```
  ### YYYY-MM-DD — balance: <調整内容>（コミットハッシュ）

  **仕様本文への影響あり（§7）**

  - <パラメータ名>: 旧値 → 新値（変更理由）
  ```

- **§7 ゲームバランス定数**（必須）: 変更した定数の値を本文テーブルに反映する
  - テーブル内の数値を更新し、コードと SPEC が常に一致するよう保つ

### 8. コミット・プッシュ・PR

`.claude/skills/_shared/commit-push-pr.md` を Read ツールで読み込み、記載の手順を実行する。コミットプレフィックスは `balance:` を使う。PR の変更概要には「パラメータ名: 旧値 → 新値（変更理由）」の形式で記載する。

## 完了時の報告

以下の形式でユーザーに報告する:

- 調整したパラメータの一覧（旧値 → 新値）
- 変更の根拠・期待される効果
- ビルド結果: ✅ 成功 / ‼️ 失敗（詳細）
- SPEC.md の更新内容（CHANGELOG.md + §7 ゲームバランス定数テーブル）
- マージ完了: PR #番号
- さらなる調整が必要と思われる箇所（あれば）
- 推奨するテスト手順（どの試合状況で効果を確認できるか）
