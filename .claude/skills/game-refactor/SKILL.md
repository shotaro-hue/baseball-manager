# Game Refactor スキル

## ワークフロー

Make a todo list for all the tasks in this workflow and work on them one after another.

### 1. 対象タスクを確認

ROADMAP.md の「保守性・コード品質」セクション（T* タグ）を読み、
未着手・部分実装のタスクを確認する。

現在の残件例（**ROADMAP.md を必ず再確認すること**）:
- **T5**: セーブデータバリデーション（ロード時 null チェック・ローリングバックアップ）
- **T6**: JSDoc 型注釈（player.js / simulation.js から順次）
- **T8**: useReducer 移行（useState + useCallback → dispatch(action) パターン化）

ユーザーが対象を指定していない場合は、ROADMAP の優先度（P0 > P1 > P2）に従って提案する。

### 2. 対象ファイルを読む

対象ファイルを読み込み、以下を把握する:
- 依存関係（props / state / callback / import の流れ）
- 変更境界（どこを変えれば外部への影響が最小になるか）
- 既存のパターン（他ファイルからどう呼ばれているか）

### 3. 改善設計

改善後の構造とその理由を書き出す。タスクタイプ別の設計方針:

| タスク | 設計で確認すること |
|---|---|
| ファイル分割 | 分割後ファイル一覧・各ファイルの責務・分割境界の妥当性 |
| JSDoc 型注釈 | 注釈を付ける関数の一覧・型定義の方針 |
| useReducer 移行 | action 型の設計・既存 useState との対応関係 |
| その他 | 変更前後の構造の差分 |

**共通原則**:
- 外部から見た動作を変えない（props・返り値・副作用を維持）
- 一度に変更するのは 1〜2 ファイルまで（差分を追いやすくする）
- ファイル削除・移動は段階的に行う

分割設計をユーザーに提示し、**必ず立ち止まって以下の3択を待つ**。承認なしに実装（ステップ4以降）へ進んではならない。

| ユーザーの反応 | 次のアクション |
|---|---|
| **承認**（「OK」「進めて」等） | ステップ4へ進む |
| **質問・修正依頼**（「この分割境界は適切？」「ファイル名を変えたい」等） | 回答・設計を修正して再提示。承認が得られるまでこのループを繰り返す |
| **中断**（「やめる」「キャンセル」等） | 実装せずに終了する |

### 4. 実装

新ファイルを作成し、分割元から該当コードを移動する。
import/export を正確に更新する。

### 5. ビルド・テスト確認

`.claude/skills/_shared/build-and-test.md` を Read ツールで読み込み、記載の手順を実行する。リファクタリングなのでビルドエラー・テスト失敗は必ず 0 にする。

### 6. ROADMAP.md を更新

完了した T* タスクのステータスを ✅ に更新する。

### 7. SPEC.md を更新

- **`CHANGELOG.md`**（必須）: 最新エントリを先頭に追加する

  ```
  ### YYYY-MM-DD — <リファクタリング内容>（コミットハッシュ）

  **仕様本文への影響あり（§2・§14）** or **仕様本文への影響なし（構造変更のみ）**

  - 分割・移動したファイルの箇条書き
  ```

- **§2 ディレクトリ構成**: 新規ファイル・ディレクトリの追加・削除があった場合のみ更新する
- **§14 保守性・技術的負債**: 完了した T* タスクのステータスを「計画中」→「✅ 完了」に更新する（必須）

### 8. コミット・プッシュ・PR

`.claude/skills/_shared/commit-push-pr.md` を Read ツールで読み込み、記載の手順を実行する。コミットプレフィックスは `refactor:` を使う。PR の変更概要には新規作成・削除・変更ファイル一覧を含める。

## 完了時の報告

- 実施したリファクタリングの概要
- 変更したファイル一覧（新規作成・削除・変更）
- ビルド結果: ✅ 成功 / ‼️ 失敗（詳細）
- ROADMAP.md 更新内容
- SPEC.md の更新内容（CHANGELOG.md + 更新したセクション一覧）
- マージ完了: PR #番号
- 次に改善を推奨する箇所（あれば）
