# Game Sprint スキル

振り返り → タスク選定 → 実装を1コマンドで完結させるオーケストレーションワークフロー。

## ワークフロー

Make a todo list for all the tasks in this workflow and work on them one after another.

### 1. モード判定

ユーザーの入力に `--review` が含まれる場合は **フルレビューモード**、それ以外は **軽量モード** で実行する。

---

### 2a. 軽量モード — タスク候補の収集

`ROADMAP.md` を読み込み、以下の優先順でタスク候補を収集する:

1. 現在進行中の Tier（🔶 部分実装）の中で未着手のタスク
2. 次の Tier（🔴 未着手）の先頭タスク
3. 既知のバグ（B* タグ）で未修正のもの

---

### 2b. フルレビューモード（`--review`）— 全体評価

`.claude/skills/game-review/SKILL.md` を Read ツールで読み込み、記載のワークフローを実行する。
game-review の完了後、評価レポートの「優先度付き次アクションリスト」をタスク候補として使用し、ステップ3に進む。

---

### 3. スプリント計画の提示（承認ゲート）

収集したタスク候補から優先度上位3件を以下の形式で提示し、**必ず立ち止まってユーザーの選択を待つ**:

```
## 今スプリントの候補タスク

| # | タイプ | タスク | 優先度 | 理由 |
|---|---|---|---|---|
| 1 | feature / bugfix / balance / ui / refactor | タスク名 | 🔴/🟡/🟢 | 選定理由 |
| 2 | ... | ... | ... | ... |
| 3 | ... | ... | ... | ... |

どれを実行しますか？（番号で選択 / 別のタスクを指定 / キャンセル）
```

| ユーザーの反応 | 次のアクション |
|---|---|
| **番号選択**（「1」「2」「3」） | 選択されたタスクでステップ4へ |
| **別タスク指定**（「〜をやりたい」） | 指定タスクの内容を確認してステップ4へ |
| **キャンセル**（「やめる」「終了」等） | スプリントを終了する |

---

### 4. 対応スキルへの委譲

選定されたタスクのタイプに応じて、対応する SKILL.md を Read ツールで読み込み、記載のワークフローを実行する:

| タスクタイプ | 委譲先 |
|---|---|
| 新機能実装 | `.claude/skills/game-feature/SKILL.md` |
| バグ修正 | `.claude/skills/game-bugfix/SKILL.md` |
| バランス調整 | `.claude/skills/game-balance/SKILL.md` |
| UI 改善 | `.claude/skills/game-ui/SKILL.md` |
| リファクタリング | `.claude/skills/game-refactor/SKILL.md` |

委譲先スキルのワークフローをそのまま実行する。**ロジックをここに重複して書かない。**

---

### 5. スプリント完了レポート

委譲先スキルの完了後、以下の形式でスプリントの総括を報告する:

```
## スプリント完了

- 実施タスク: <タスク名>（<タイプ>）
- 変更ファイル: <一覧>
- ビルド・テスト: ✅ 成功 / ‼️ 失敗
- マージ: PR #<番号>

## 次スプリントの推奨タスク

1. <タスク名> — <理由>
2. <タスク名> — <理由>
```
